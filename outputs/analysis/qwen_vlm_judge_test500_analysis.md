# Qwen VLM Judge Test500 Analysis

## Setup

Dataset: `data/pairs/annotations/fcdb_3way_test.jsonl`

Samples: 500 full test pairs

Model: Qwen VLM judge via OpenAI-compatible API

Prompt style: source image first, edited/reframed image second, ask whether the edited image improves composition.

## Interpretation

The main failure mode is prompt/order bias.

The current prompt tells Qwen that image 1 is the `source image` and image 2 is the `edited/reframed image`, then asks whether the edited image improves composition. In FCDB, however, both images are just candidate crops from the same original photo. The second image is not necessarily generated by a real editor and is not necessarily intended to be better.

Qwen appears to assume that the second image is a deliberate improvement. Its reasons often use language such as `tightens the framing`, `centering the subject`, and `reducing empty space`, even on samples where FCDB voters preferred the first crop.

This creates a strong one-sided bias:

- It has high recall on `win` because it often says the second image is better.
- It has very low recall on `lose` and `tie` because it rarely rejects the edited image.
- Macro F1 is poor despite reasonable-looking explanations.

## Takeaway

The current Qwen result should not be treated as a fair VLM upper bound. It is mostly measuring how strongly the prompt biases the model toward the second image.

A better VLM protocol should avoid source/edit wording and instead use a blind pairwise comparison:

```text
You will see two candidate crops A and B from the same original photo.
Which crop has better photographic composition?
Return A, B, or tie.
Do not assume either image is edited or intended to be better.
```

Then map:

- `B` -> `win`
- `A` -> `lose`
- `tie` -> `tie`

For even stronger control, randomize order per sample and optionally run both orders to measure order consistency.

## Recommended Next Step

Build Baseline 7: blind Qwen A/B judge.

Required changes:

1. Rename images in prompt to `Candidate A` and `Candidate B`.
2. Remove all source/edit/improvement wording.
3. Ask for `choice: A|B|tie` plus scores/reason.
4. Randomize whether source/edit is shown as A/B and map the answer back.
5. Report order-bias diagnostics: predicted A/B/tie distribution and optional flip consistency.
